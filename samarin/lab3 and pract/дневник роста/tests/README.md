# Тесты Selenium для Дневника растений

Один файл с комплексными тестами, покрывающими фронтенд, бэкенд и базу данных.

## Структура

- `test_all.py` - Все тесты в одном файле
- `requirements.txt` - Зависимости

## Установка

```bash
pip install -r tests/requirements.txt
```

## Запуск тестов

```bash
# Запуск всех тестов
pytest tests/test_all.py -v

# Запуск с подробным выводом
pytest tests/test_all.py -v -s
```

## Покрытие тестов

### 1. Регистрация и вход (test_01)
- Регистрация нового пользователя
- Вход в систему
- Проверка отображения имени пользователя

### 2. Создание растения (test_02)
- Навигация к форме добавления
- Заполнение и отправка формы
- Проверка создания в списке

### 3. Просмотр деталей растения (test_03)
- Создание растения
- Переход к деталям
- Проверка отображения информации

### 4. Добавление записи дневника (test_04)
- Создание растения
- Добавление записи через API
- Проверка отображения записи

### 5. Загрузка фотографии (test_05)
- Создание растения
- Переход на вкладку фото
- Загрузка тестового изображения
- Проверка загрузки

### 6. Создание напоминания (test_06)
- Создание растения
- Создание напоминания через API
- Проверка отображения напоминания

### 7. Навигация (test_07)
- Проверка всех кнопок боковой панели
- Переключение между разделами

### 8. Выход из системы (test_08)
- Выход из системы
- Проверка очистки localStorage
- Возврат на форму авторизации

## Особенности

- **Полная изоляция**: каждый тест открывает и закрывает браузер отдельно
- **Автоматическая очистка**: localStorage очищается перед каждым тестом
- **Уникальные пользователи**: каждый тест создает уникального пользователя
- **Покрытие всего функционала**: фронтенд, бэкенд API, база данных

## Требования

1. Приложение должно быть запущено на `http://localhost`
2. Backend API должен быть доступен на `http://localhost:8000`
3. Chrome браузер должен быть установлен

## Настройка URL

```bash
export BASE_URL=http://your-url
pytest tests/test_all.py -v
```
